Задание 1: "Smart Search & Multi-Filter" (Avia Focus)
Контекст: В авиа-доменах поиск возвращает массив offers с разными комбинациями перелетов, багажа и правил тарифа.
Техническое задание:
Интеграция: Реализовать экран результатов поиска. Данные запрашиваются через TanStack Query.
Логика фильтрации: Реализовать сложный клиентский фильтр (без повторного запроса к API):
По количеству пересадок (0, 1, 2+).
По авиакомпаниям (динамический список из текущей выдачи).
По наличию багажа (is_baggage_included).
Состояние: Применить Zustand для синхронизации выбранных фильтров между экраном поиска и модальным окном фильтров.
Performance: Использовать useMemo для тяжелых вычислений фильтрации, чтобы UI не фризил при переключении чекбоксов.
Критерий Middle+: Умение работать с нормализацией данных (если один и тот же сегмент полета встречается в разных офферах) и грамотная обработка состояния "Ничего не найдено".

Задание 2: "Interactive Wagon Map" (Railway Focus)
Контекст: В ЖД API (/travel/) при выборе поезда возвращается схема вагона с типами мест (верхние, нижние, боковые) и их статусом (available, taken).
Техническое задание:
Визуализация: Отрисовать интерактивную карту вагона.
Карта должна поддерживать Zoom и Pan (через react-native-reanimated или специализированные библиотеки).
Места должны быть кликабельными.
Бизнес-логика:
Реализовать ограничение: пользователь не может выбрать более 4-х мест за раз.
Автоматический расчет стоимости: при клике на "нижнее" или "верхнее" цена должна обновляться мгновенно (данные брать из объекта carriages в API).
UI Feedback: При выборе места использовать Haptics (виброотклик) и менять стиль места плавной анимацией.
Критерий Middle+: Работа с SVG или сложной сеткой (Grid/Absolute), предотвращение лишних ререндеров всех 54+ мест при выборе одного.

Задание 3: "Payment Flow & Deep Link Resilience" (Infrastructure Focus)
Контекст: Работа с IOKA SDK или API требует создания заказа (createOrder) и последующей обработки 3DS.
Техническое задание:
Checkout Flow: Реализовать переход из корзины (бронирования) на экран оплаты.
Использовать Expo Router для навигации.
Реализовать "Interceptor" для API: если токен сессии оплаты истек (401/403), нужно автоматически инициировать рефреш или возврат на шаг создания заказа.
Deep Linking: Настроить обработку success_url и failure_url. Приложение должно корректно перехватывать возврат из банковского приложения (3DS) и открывать нужный экран в стеке (tabs)/orders/[id].
Reliability: Интегрировать Sentry. Настроить Sentry.captureException для специфических ошибок API Ioka (например, когда amount не совпадает или транзакция отклонена), прикрепляя order_id как тег.
Критерий Middle+: Настройка EAS Build конфигов для разных схем (dev/prod) и знание того, как работает Linking.useURL() внутри Expo Router для обработки внешних переходов.
